## Product Positioning

Talordata SERP API 帮助开发者、SEO 团队、数据团队和 AI 产品团队，通过结构化 API 获取实时搜索引擎结果数据。

产品定位应围绕以下方向展开：

- 获取实时搜索引擎数据。
- 输出结构化 SERP 数据。
- 降低用户自行搭建和维护搜索数据采集系统的复杂度。
- 支持全球和本地化搜索结果采集。
- 支持 SEO 监控、排名追踪、竞品情报、本地 SEO、电商监控和 AI / LLM 工作流。
- 减轻用户在搜索结果解析、地理位置限制、验证码问题和不稳定抓取流程上的工程负担。
- 强调开发者友好集成，包括清晰参数、稳定输出和实用示例。

## Core Product Value Propositions

- 通过 API 获取实时 SERP 数据。
- 支持多种搜索结果类型和搜索引擎。
- 返回结构化数据，可用于仪表盘、SEO 工具、数据管道和 AI 工作流。
- 在适用场景下，支持按国家、地区、城市或语言采集本地化搜索数据。
- 帮助用户降低维护爬虫、解析 HTML、处理地理位置限制和验证码中断的复杂度。
- 适用于搜索监控、市场情报、AI grounding、RAG 数据增强、竞品追踪和电商可见性分析。
- 在合适场景下，应强调开发者友好的文档和 API 示例。
- 商业表达应清晰、实用，不夸张。

## Forbidden / Caution Rules

- 除非我明确要求，不要提及被禁止或无关的旧产品名称。
- 不要虚构产品能力。
- 不要虚构统计数据、成功率、客户案例或基准测试结果。
- 不要过度承诺结果，例如保证排名、保证流量或保证收录。
- 不要在教育型博客文章中写硬广。
- 不要以看起来像 spam 的方式过度使用竞品名称。
- 除非明确要求，不要在最终可发布正文中添加外部链接。
- 不要对绕过限制做不安全或夸张的表达。
- 不要让文章听起来像通用 AI 模板。
- 不要让每篇文章都使用重复结构。
- 不要让每篇文章都用相同的定义式开头。
- 不要为了增加篇幅而添加不必要章节。
