# Writing Modes

## Purpose

Writer should choose a writing mode based on the selected workflow and user intent.

This file is generic. Workflow files decide which mode to use and what output structure is required.

## Available Modes

| Mode | Best For |
|---|---|
| Blog Writer | Educational or SEO/GEO blog content |
| Landing Page Copywriter | Product/topic/service pages |
| Help Center Writer | Documentation and how-to support content |
| Comparison Page Writer | Alternative, versus, and buyer decision content |
| Integration Tutorial Writer | Step-by-step tool integration guides |
| Report Writer | Analytical reports and business summaries |
| Short Copy Writer | Ads, social posts, UI microcopy, email snippets |
| Rewrite Editor | Improving existing drafts |
| Humanizer | Reducing AI tone and improving naturalness |

## 1. Blog Writer

Use when the workflow requires educational or search-driven long-form content.

Core behaviors:

- Start with search/user intent.
- Choose a structure pattern.
- Create a useful opening.
- Build sections around user questions and decision criteria.
- Avoid generic filler.
- Make product mentions natural.

## 2. Landing Page Copywriter

Use for product pages, topic pages, and conversion pages.

Core behaviors:

- Lead with user pain and product value.
- Use clear hero copy.
- Group benefits into scannable modules.
- Include proof, use cases, and CTA logic.
- Avoid blog-style long explanations.

## 3. Help Center Writer

Use for documentation and support articles.

Core behaviors:

- Write task-first instructions.
- Use steps, examples, parameters, and expected results.
- Avoid marketing language.
- Include troubleshooting where useful.
- Keep sentences direct.

## 4. Comparison Page Writer

Use for alternatives, comparisons, and decision pages.

Core behaviors:

- Provide a direct comparison early.
- Use tables when useful.
- Explain when each option fits.
- Avoid unfair competitor claims.
- Include decision criteria.

## 5. Integration Tutorial Writer

Use for setup guides and tool integrations.

Core behaviors:

- State prerequisites.
- Provide step-by-step flow.
- Include configuration details.
- Note common errors.
- Provide final verification steps.

## 6. Report Writer

Use for analysis reports.

Core behaviors:

- Separate facts, analysis, and recommendations.
- Use executive summaries.
- Make risks and assumptions visible.
- Provide decision-oriented conclusions.

## 7. Short Copy Writer

Use for short-form copy.

Core behaviors:

- One idea per line or block.
- Strong clarity.
- No over-explaining.
- Match channel constraints.

## 8. Rewrite Editor

Use when improving an existing draft.

Core behaviors:

- Preserve confirmed meaning.
- Improve clarity, rhythm, and structure.
- Do not add unsupported claims.
- Record what changed.

## 9. Humanizer

Use when content feels AI-generated.

Core behaviors:

- Remove generic transitions.
- Vary sentence rhythm.
- Replace inflated language.
- Cut filler.
- Preserve technical accuracy.
