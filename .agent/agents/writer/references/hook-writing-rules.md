# Hook Writing Rules

## Purpose

This file helps Writer create stronger openings for articles, pages, reports, and other written artifacts.

A good hook should help the reader understand why the content matters without sounding dramatic or AI-generated.

## Core Rule

The opening should answer one of these quickly:

- What problem does this solve?
- Who is this for?
- What decision does this help with?
- What will the reader know or be able to do after reading?
- Why is this topic not as simple as it looks?

## Hook Types

### 1. Direct Answer Hook

Use when the user likely wants a quick answer.

Pattern:

```text
A [topic] helps [audience] do [job] without [pain]. This guide explains [scope].
```

### 2. Problem Hook

Use when the topic is pain-driven.

Pattern:

```text
Teams often try to solve [problem] with [common approach], but run into [pain]. A better workflow starts with [key idea].
```

### 3. Decision Hook

Use when the article helps readers choose.

Pattern:

```text
The right choice depends on [criteria]. This article compares [options] by [decision factors].
```

### 4. Workflow Hook

Use when the content explains a process.

Pattern:

```text
The workflow is simple in theory: [step]. In practice, teams also need to handle [real constraints].
```

### 5. Contrarian Hook

Use when the obvious framing is weak.

Pattern:

```text
The question is not only [surface question]. The harder part is [deeper problem].
```

## Weak Hooks to Avoid

Avoid openings that:

- Start with a broad history lesson
- Define a term everyone already understands
- Say “In today’s digital landscape”
- Say “As businesses increasingly...”
- Say “In the ever-evolving world...”
- Use generic claims without a concrete problem
- Sound like a product brochure
- Delay the answer too long

## AI-Like Hook Patterns to Avoid

Avoid:

```text
In today's fast-paced digital world...
It is crucial to understand...
This comprehensive guide will delve into...
Whether you're a beginner or an expert...
Unlock the power of...
```

## Hook Self-Check

Before finalizing an opening, ask:

- Does it answer the reader's intent quickly?
- Is it specific to this topic?
- Could it appear in any generic article?
- Does it overpromise?
- Is it natural enough for a human editor?
- Does it make the next section easy to read?
