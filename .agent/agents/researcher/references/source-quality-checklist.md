# Source Quality Checklist

## Purpose

This checklist helps Researcher decide whether a source is useful enough to include in a research report or source index.

## Include a Source Only If It Helps With At Least One

- Verify a factual claim
- Understand user intent
- Understand target audience language
- Identify competitor structure
- Identify content gaps
- Support a comparison
- Clarify product positioning
- Provide examples
- Provide terminology used by real users
- Confirm current documentation or feature behavior
- Support a workflow-specific quality standard

## Scoring

Use this simple scale:

```text
High = directly useful and trustworthy
Medium = useful but needs caution
Low = weak support or noisy
Reject = do not include
```

## High-Value Sources

Usually high-value:

- Official documentation
- First-party product pages
- Clear competitor pages
- Reputable industry publications
- Current help center docs
- Search result pages when the workflow requires SERP pattern analysis
- User-provided internal material

## Medium-Value Sources

May be useful:

- Blog posts from known companies
- Comparison pages
- Product review articles
- Community discussions, if used carefully
- Older articles with still-valid conceptual content

## Low-Value Sources

Use cautiously:

- Undated articles
- Thin affiliate pages
- AI-generated-looking pages
- Pages full of generic claims
- Aggregators with no clear source
- Outdated feature lists
- Broken or inaccessible pages

## Reject Sources

Reject sources that:

- Cannot be verified
- Are irrelevant to the task
- Only repeat generic claims
- Lack enough content
- Contradict known product facts without evidence
- Are included only to make the report look longer

## Source Index Anti-Patterns

Do not:

- Add ten sources when three strong ones are enough.
- Treat competitor marketing claims as objective facts.
- Add source URLs without explaining why they matter.
- Use a source for a section it does not support.
- Hide uncertainty.
