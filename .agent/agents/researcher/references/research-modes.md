# Research Modes

## Purpose

Researcher should not use the same depth for every task. Select a research mode based on the selected workflow, task risk, available context, and downstream needs.

## Mode Selection Matrix

| Mode | Use When | Output Depth | Cost |
|---|---|---:|---:|
| Quick Research | Low-risk task, enough existing context | Light | Low |
| Source Index Research | Writer needs sources, not long analysis | Medium | Medium |
| SERP Pattern Research | SEO/GEO or search-intent-driven content | Medium | Medium |
| Competitor Page Teardown | Landing pages, product pages, comparison pages | Deep | Medium-High |
| Deep Research | High-value strategic content | Deep | High |
| Recursive Research | Complex unknown problem | Very Deep | Very High |

## 1. Quick Research

Use for:

- Simple writing tasks
- Short copy
- Small updates
- Tasks with enough existing context

Must produce:

- Short summary
- Key assumptions
- Missing information
- Recommended next step

Avoid:

- Long competitor reports
- Over-collecting sources
- Over-explaining obvious background

## 2. Source Index Research

Use when downstream Writer needs reliable material.

Must produce:

- Source Index
- Why each source matters
- Recommended use for each source
- Source confidence
- Any local extracted file path, if available

Use the standard Source Index fields from `source-extraction-rules.md`:

```markdown
| ID | Source Name | URL / Local File | Type | Confidence | Why Useful | Recommended Use | Cautions |
|---|---|---|---|---|---|---|---|
```

## 3. SERP Pattern Research

Use when the workflow requires search intent or SEO/GEO strategy.

Must analyze:

- Search intent
- Target audience
- Common SERP page patterns
- Competitor section patterns
- Content gap
- Differentiation angle
- Questions users likely ask
- Risks or unsupported assumptions

Must not:

- Invent keyword volume
- Invent ranking difficulty
- Copy competitor wording
- Over-index on one competitor

## 4. Competitor Page Teardown

Use for:

- Topic pages
- Landing pages
- Product pages
- Integration pages
- Alternative/comparison pages

Must analyze:

- H1/H2/H3 structure
- Conversion blocks
- CTA placement
- Product claims
- Use cases
- Pricing/trial messaging, if relevant
- FAQ
- Schema or structured content hints
- Differentiation opportunities

Must output:

- What competitors do repeatedly
- What they miss
- What the user can do differently
- Which patterns should be borrowed
- Which patterns should be avoided

## 5. Deep Research

Use for high-value pieces where accuracy and strategy matter.

Must include:

- Multiple source types
- Competing viewpoints, if relevant
- Evidence strength
- Knowledge gaps
- Recommendation confidence

Do not use Deep Research for routine tasks unless requested.

## 6. Recursive Research

Use only when the problem is broad, undefined, or cross-domain.

Process:

1. Break the problem into sub-questions.
2. Research each sub-question.
3. Identify contradictions and gaps.
4. Create a synthesis.
5. Recommend the next workflow action.

## Research Output Quality Bar

A good research output should answer:

- What is the user really trying to achieve?
- What does the audience need?
- What facts are reliable?
- What is uncertain?
- What should the Writer do next?
- What should the Reviewer watch for?
