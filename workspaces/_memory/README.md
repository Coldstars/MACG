# Workspace Memory

## Purpose

This directory stores reusable memory learned from real tasks.

Use memory files to improve future work, but do not treat them as stronger than:

1. `AGENTS.md`
2. `profile.md`
3. `.agent/rules/`
4. selected workflow files
5. role `SKILL.md` files
6. current workspace `process.md`

## Directory Map

### `content/`

Reusable lessons for content production tasks, including blogs, SEO/GEO articles, product pages, landing pages, tutorials, comparison pages, and help-center content.

Files:

- `approved-patterns.md`: Writing structures, openings, tones, examples, and content patterns the user liked.
- `rejected-patterns.md`: Writing patterns, structures, tones, or output formats the user disliked and wants to avoid.
- `content-lessons.md`: Reusable content strategy lessons confirmed by the user.

Relevant content workflows should explicitly read these files when producing or revising content.

### `product/`

Reusable product and messaging lessons learned from tasks.

Use it for:

- Product positioning refinements.
- Feature wording cautions.
- Reusable value proposition notes.
- Confirmed product capability boundaries.
- Messaging rules for specific audiences or use cases.

This directory does not replace `profile.md`.

### `workflow/`

Reusable lessons about how the local multi-agent workflow should operate.

Use it for:

- Process improvements.
- Agent handoff lessons.
- Workspace file structure preferences.
- Human review gate preferences.
- Repeated operational mistakes to avoid.

This directory does not override `AGENTS.md`, `.agent/rules/`, selected workflow files, or role `SKILL.md` files.

## Update Rule

Only add or change memory when the user explicitly confirms a preference, correction, or lesson should be remembered.
